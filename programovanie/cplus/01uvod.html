Úvod do jazyka

Jazyk C
patrí ku nižším programovým jazykom (je vhodný aj na systémové úlohy) je spojený s operačným systémom UNIX, ktorý je v ňom celý napísaný a aj jeho príkazy sú C-čku podobné.C++ je nastavba jazyka C, ktorá umožňuje písať objektovo-orientované programy.
Výhody
- je nezávislý na platforme, zdrojáky sú prenosné medzi rôznymi OS. Často sa aj ladí na inom OS ako sa potom spúšťa.
- malé množstvo príkazov -> jednoduchý, rýchly kompilátor
- C++ je flexibilný jazyk - umožňuje prekrývanie funkcií i operátorov
Nevýhody
- ťažšie čitatelné programy (jeden príkaz má viac efektov)
- je užívateľsky menej príjemný a je určený hlavne pre profesionálov
Vlastnosti, odlišnosti od pascalu
- rozlišuje malé a veľké písmená podobne ako LINUX
- nemá hlavný program ako je v pascale
- celý program sa skladá len z funkcií, ktoré sa nemôžu vnárať
- bodkočiarka nie je oddelovač príkazov ale je jeho súčasť
- stringy sú reprezentované odlišne
- práca so smerníkmi je podstatne vylepšená
Stavba programu
metapríkazy
direktívy pre prekladač #include - ako uses v pascale, #define - na def. konštánt, makier
definovanie globálnych typov
typedef- podobné ako TYPE v pascale
definovanie globálnych premenných
< typ> zoznam_premenných ;
definovanie funkcií
skladá sa z hlavičky a bloku. Jedna funkcia musí mať meno main Každý program v jazyku C sa skladá z jednej alebo viacerých funkcií
Stavba funkcie
hlavička
začína určením typu a za medzerou nasleduje názov funkcie. Za názvom sú vždy okrúhle zátvorky v ktorých môžu byť deklarované parametre. Za hlavičkou nikdy nie jebodkočiarka
blok
telo funkcie. Tu možno opäť zadať metapríkazy, zadeklarovať lokálne typy, lokálne konštanty, lokálne makrá, lokálne premenné. Na rozdiel od pascalu môžu byť deklarácie aj vnútri tela (medzi príkazmi). Nie je možné deklarovať podprogram.
Prvý program
Na rozdiel od pascalu nie sú príkazy vstupu a výstupu štandartnými príkazmi jazyka a sú v knižniciach, ktoré musíme pripojiť ku nášmu programu (podobne ako v pascale príkaz uses crt).
Prvy.cpp
  #include <iostream>         //pripojenie kniznice
  #define POCET 5                
    //zadefinuje konstantu POCET s hodnotou 5 (nedavame ;), za 5 je hned enter
  #typedef integer int;      //definovanie noveho typu integer
  integer sucet;             //vytvori celociselnu premennu sucet
  void pripocitaj(int);      //dopredna deklaracia funkcie (podobne ako forward v pascale)
  void main(void)            //hlavicka "hlavneho podprogramu" - bez bodkociarky
  {                          //zaciatok bloku
   integer c,i;              //lokalne premenne
   for (i=1;i<=POCET; i++)   //cyklus
   {                         //zacina zlozeny prikaz 
      cout <<"Zadaj "<<i<<"-te cislo:";   //napise text na obrazovku
      cin >>c;        //nacita z klavesnice 
      pripocitaj(c);	     //zavola podprogram s parametrom c
   }
   cout <<"Vysledny sucet = "<<sucet<< endl;
  }
  void pripocitaj(int cislo)  //hlavicka podprogramu
  { sucet+=cislo; }

Základné prvky jazyka
Ako v každom programovacom jazyku i tu je zdrojový program tvorený postupnosťou znakov. Prípustné sú znaky:
písmená: A-Z, a-z ... program rozlišuje malé a veľké písmená
číslice: 0-9
špec. znaky: + - * / % ... aritmetické operátory (% je ako mod)
[] ... na indexy poľa
() ... zátvorky funkcií, arimetické zátvorky, cykly
{} ... určenie bloku, zložený príkaz
= ... operátor priradenia
<> ... porovnávacie operátory
. ... na prístup ku položkám záznamu
& ... zistenie adresy premennej (ako operátor @ v pascale)
! ... na negáciu
~ ... na bitovú negáciu
^ ... na bitové XOR
| ... na logické a binárne OR
% ... na operáciu MOD
# ... metapríkazy
" ... na textové konštanty (narozdiel od ' pascalských)
' ... na znakové konštanty
; ... napísanie príkazov, for cyklu
, ... operátor čiarka (oddelovanie výrazov)
? ... operátor otáznik
\ ... na zadávanie špeciálnych znakov

Prostredie
V Borland C++ je IDE prostredie podobné ako v pascale so zabudovaným kompilátorom a debugerom. Rozdiel od pascalu je v tom, že kompiluje na disk do aktuálneho adresára (implicitne bin), kde vytvorí súbory meno.obj, a meno.exe. Preto je vhodnejšie pred prekladom nastaviť aktuálny adresár do svojho adresára a tam aj zapísať svoj zdroják aby tieto súbory boli pokope. Ak chceme vytvárať programy *.com, musíme siahnuť po externom prekladači.
Pod UNIXom nemáme zakúpené žiadne prívetivé prostredie a preto postup bude taký, že napíšeme a odladíme program pod DOSom (samoyrejme používame len objekty dostupné jak pod DOSom tak pod UNIXom), po odladení si ho prenesieme na LINUX a tam ho preložíme pomocou príkazu gcc meno.cpp. Ak sme šikovný vznikne súbor a.out. Potom si a.out premenujeme na požadované meno. Spustenie takéhoto programu pod UNIXom je nasledovné: 1. súbor má nastavené práva x (spustenie) - stačí zadať ./a.out 2. súbor nemá nastavené práva x - zadáme bash ./a.out //musíme zavolať bash, ktorý nam ho spustí.
